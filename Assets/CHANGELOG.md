# 更新日志

* 新增回复生成完成时的通知信息发送, 点击后即可唤回窗口至前台
* 将 Markdown 文本渲染改为 1 秒一次, 避免在长文本下造成过大性能负担
* 重构 SQLite 访问链路, 通过有界单连接调度器串行处理前台与维护任务, 并启用 WAL、忙等待和受控页缓存
* 新增事件、记忆、人物与关系复合索引, 为记忆内容和标签建立 trigram 全文索引并自动同步增删改
* 为对话、人物和记忆补充稳定游标分页查询, 降低大体量数据读取时的单次查询与物化压力
* 将每轮场景、状态、指令、人物状态和关系读取整合为一致性快照, 减少重复查询和 N+1 访问
* 将回合业务写入与回滚审计收拢到同一事务, 避免业务数据与审计记录状态不一致
* 优化知识和记忆向量索引, 仅分批修复内容或模型指纹变化的条目, 未变化内容不再重复生成向量
* 优化知识和记忆召回, 每轮共享一次查询向量, 使用有界候选集、会话与时间线过滤及批量实体读取
* 为历史上下文增加最近 40 轮和 12000 估算词元预算, 超出范围后增量维护场景进展摘要
* 缓存相同配置的聊天与向量客户端, Release 模式关闭配置及语言热重载
* 主界面右侧边栏“记忆”与“人物”面板中各卡片改为默认关闭状态
